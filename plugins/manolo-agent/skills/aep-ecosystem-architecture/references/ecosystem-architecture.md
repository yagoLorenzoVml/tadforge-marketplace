# AEP Ecosystem Architecture

Produce an evidence-backed map covering:

- Source system and owner.
- Data domain and business purpose.
- Connection, batch or streaming mode, cadence, volume, latency, and region.
- XDM schema, target dataset, validation, and retention.
- Identity namespaces, Profile enablement, and merge policy.
- Audiences, queries, models, or other consumers.
- Destinations, mappings, schedule, and activation SLA.
- Consent, labels, policies, deletion, and residency controls.
- Monitoring, reconciliation, incident owner, and recovery path.

Highlight single points of failure, ambiguous ownership, duplicate data paths, latency mismatches, hidden manual steps, identity inconsistencies, governance gaps, and environments without promotion discipline.

Use diagrams only after the inventory is evidence-backed. Mark unknown nodes and interfaces explicitly instead of completing the picture from assumption.
