# AEP Sandbox Audit

## Scope

State organization if available, sandbox name and type, audit timestamp, accessible capabilities, inaccessible areas, and whether findings are exhaustive or sampled.

## Report

1. Executive health: overall status, critical risks, and immediate actions.
2. Inventory: schemas, field groups, datasets, namespaces, merge policies, audiences, sources, dataflows, and destinations.
3. Data model: duplicates, weak semantics, incompatible types, undocumented extensions, and Profile enablement.
4. Identity and Profile: namespace quality, graph risks, fragments, and merge policies.
5. Ingestion: failed or stale flows, error trends, volume anomalies, and freshness.
6. Audiences and activation: stale or duplicate audiences, evaluation issues, mapping risks, and reconciliation.
7. Governance: labels, policy enforcement, consent, minimization, retention, and deletion gaps visible to the tools.
8. Operations: ownership, naming, lifecycle, monitoring, promotion, and runbook gaps.
9. Prioritized remediation: severity, evidence, impact, action, dependency, owner if known, and target date if known.

Use `Unknown` for inaccessible evidence and explain how to obtain it. Do not infer compliance merely because no violation is visible.
